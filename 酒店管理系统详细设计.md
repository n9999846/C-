# 酒店客房管理系统 — 详细设计文档

## 1. 项目概述

本项目是一个基于C++面向对象程序设计的酒店客房管理系统，支持房型管理、客户管理、预订/入住/退房流程、账单计费、营收统计等功能。系统采用Qt实现图形界面（由AI辅助生成），核心OOP逻辑由四位成员分别完成。

**核心设计目标：**
- 每位成员都有独立的类设计任务，覆盖C++ OOP主要知识点
- 通过继承、多态、运算符重载、异常处理、STL、单例模式等体现设计能力
- 各模块之间通过指针/引用交互，形成完整的酒店业务流程

## 3. 成员A：房间体系（Room Hierarchy）

成员A负责设计酒店所有房间相关的类。这是整个系统的数据基础，BookingManager和BillingManager都需要依赖房间信息。

### 3.1 `Room` 抽象基类

**作用：** 定义所有房间类型的公共接口和公共属性。无论标准间、豪华间还是套房，都有房间号、楼层、价格、状态这些共性。

**为什么设计为抽象类：**
因为"Room"本身是一个抽象概念，系统中实际存在的都是具体房型。通过抽象基类，可以统一用 `Room*` 指针管理不同类型的房间对象，实现多态。

**核心成员：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `roomNumber` | `string` | 房间号，唯一标识，如"101" |
| `floor` | `int` | 楼层 |
| `pricePerNight` | `double` | 每晚基础价格 |
| `status` | `enum Status` | 房间状态：AVAILABLE/BOOKED/OCCUPIED/MAINTENANCE |

**核心方法：**

```cpp
// 纯虚函数：子类必须实现
virtual string getRoomType() const = 0;        // 返回房型名称
virtual int getMaxGuests() const = 0;          // 返回最大入住人数
virtual string getDescription() const = 0;     // 返回房间描述

// 普通虚函数：子类可选择覆盖
virtual double getActualPrice() const;         // 默认返回pricePerNight

// 状态转换方法
void book();           // AVAILABLE -> BOOKED
void checkIn();        // BOOKED -> OCCUPIED
void checkOut();       // OCCUPIED -> AVAILABLE
void setMaintenance(); // 任意状态 -> MAINTENANCE
```

**运算符重载设计：**
- `operator<`：按价格排序，配合STL sort使用
- `operator==`：按房间号比较，判断是否是同一个房间
- `operator<<`：格式化输出房间信息到控制台或日志

**注意点：**
- 析构函数必须声明为虚函数，否则 `delete Room*` 时无法正确释放子类对象
- 状态转换需要检查当前状态是否合法，例如不能从AVAILABLE直接变为OCCUPIED

### 3.2 `StandardRoom` 标准间类

**作用：** 继承自Room，表示酒店最基础房型。

**新增属性：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `bedCount` | `int` | 床位数，1或2 |
| `hasWindow` | `bool` | 是否有窗 |
| `hasWifi` | `bool` | 是否含WiFi |

**方法覆盖：**
- `getRoomType()` 返回 `"标准间"`
- `getMaxGuests()` 返回 `bedCount + 1`，例如1张床住2人，2张床住3人
- `getDescription()` 返回类似 `"标准间，2张床，有窗，含WiFi"` 的描述

**设计说明：**
标准间是酒店数量最多的房型，属性简单，适合作为第一个具体子类实现，逻辑直观。

---

### 3.3 `DeluxeRoom` 豪华间类

**作用：** 比标准间更高级的房型，带额外设施。

**新增属性：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `hasBalcony` | `bool` | 是否有阳台 |
| `hasBathtub` | `bool` | 是否有浴缸 |
| `hasBreakfast` | `bool` | 是否含早餐 |

**方法覆盖：**
- `getRoomType()` 返回 `"豪华间"`
- `getMaxGuests()` 返回 `3`
- `getActualPrice()` 可覆盖：如果含早餐，价格可能上浮

**设计说明：**
豪华间作为套房和套房之间的中间层，为二次继承做铺垫。DeluxeRoom本身继承自Room，而SuiteRoom又继承自DeluxeRoom，形成继承链。

---

### 3.4 `SuiteRoom` 套房类（二次继承）

**作用：** 酒店最高级房型，继承DeluxeRoom的所有设施，并增加独立客厅、会议室等。

**继承链：** `Room → DeluxeRoom → SuiteRoom`

**新增属性：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `livingRoomArea` | `double` | 客厅面积（平方米） |
| `hasMeetingRoom` | `bool` | 是否带会议室 |
| `hasKitchen` | `bool` | 是否带厨房 |

**方法覆盖：**
- `getRoomType()` 返回 `"套房"`
- `getMaxGuests()` 返回 `4`
- `getDescription()` 在豪华间描述基础上追加套房专属信息
- `getActualPrice()` 返回基础价格的1.5~2倍，体现套房溢价

**设计说明：**
这是系统中最重要的继承结构之一。二次继承展示了对继承层次的理解。套房既有豪华间的阳台/浴缸/早餐，又有自己的客厅/会议室/厨房。

---

### 3.5 `RoomManager` 房间管理器

**作用：** 统一管理所有房间对象。对上层模块（如BookingManager）屏蔽具体的房间类型差异，只暴露统一的 `Room*` 接口。

**内部容器：**
```cpp
vector<Room*> rooms;
```
使用 `Room*` 指针可以存储 `StandardRoom`、`DeluxeRoom`、`SuiteRoom` 三种对象，体现多态。

**核心方法分类：**

#### 增删改查
- `addRoom(Room* room)`：添加新房间
- `removeRoom(const string& roomNumber)`：删除房间
- `findRoom(const string& roomNumber)`：按房间号查找
- `getAllRooms()`：获取所有房间
- `getAvailableRooms()`：获取所有空闲房间
- `getRoomsByType(const string& type)`：按房型筛选
- `getRoomsByFloor(int floor)`：按楼层筛选
- `getRoomsByPriceRange(double min, double max)`：按价格区间筛选

#### 排序
- `sortByPrice()`：使用 `std::sort` + lambda 按价格升序
- `sortByRoomNumber()`：按房间号升序
- `sortByFloor()`：按楼层升序

#### 状态操作
- `bookRoom(roomNumber)`：将房间状态改为BOOKED
- `occupyRoom(roomNumber)`：将房间状态改为OCCUPIED
- `releaseRoom(roomNumber)`：将房间状态改回AVAILABLE
- `setRoomMaintenance(roomNumber)`：将房间状态改为MAINTENANCE

#### 统计
- `getOccupancyRate()`：入住率 = 已入住房间数 / 总房间数
- `getRoomTypeDistribution()`：返回各房型数量，使用 `map<string, int>`
- `getAveragePriceByType()`：返回各房型均价

#### 文件IO
- `saveRooms(filename)`：将所有房间序列化到文件
- `loadRooms(filename)`：从文件反序列化创建房间对象

**内存管理注意点：**
`RoomManager` 析构函数必须遍历 `vector<Room*>`，逐个 `delete`，否则会造成内存泄漏。

## 4. 成员B：客户与预订体系

成员B负责酒店客户管理和最核心的预订业务流程。这是系统业务逻辑最密集的部分。

### 4.1 `Customer` 客户基类

**作用：** 表示酒店普通客户，封装客户的基本信息和借阅相关行为。

**核心成员：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 客户唯一编号，如"C001" |
| `name` | `string` | 客户姓名 |
| `phone` | `string` | 手机号 |
| `idCard` | `string` | 身份证号 |
| `totalSpent` | `int` | 累计消费金额 |

**虚函数设计：**

```cpp
virtual string getCustomerType() const;       // 返回 "普通客户"
virtual double getDiscountRate() const;       // 返回 1.0（无折扣）
virtual int getMaxActiveBookings() const;     // 返回 3（最多3个活跃预订）
```

这些函数在基类中提供默认实现，VIP子类中可以覆盖以改变行为。这就是**多态**的核心应用场景。

**运算符重载：**
- `operator<<`：输出客户信息
- `operator==`：按id比较两个客户

**设计说明：**
将 `getDiscountRate()` 和 `getMaxActiveBookings()` 设计为虚函数，是因为不同类型的客户（普通/VIP）在这些行为上有差异，但调用者可以统一用 `Customer*` 处理。

---

### 4.2 `VIPCustomer` VIP客户类（二次继承）

**作用：** 表示酒店的VIP客户，享有更高的预订上限和更低的折扣。

**继承链：** 虽然 Customer 不是抽象类，但VIP客户继承Customer后可以覆盖虚函数，实现差异化行为。如果项目中 Person 作为抽象基类，则继承链为 `Person → Customer → VIPCustomer`。

**新增属性：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `vipLevel` | `int` | VIP等级，1/2/3 |
| `memberSince` | `string` | 成为VIP的日期 |

**方法覆盖：**

```cpp
string getCustomerType() const override { return "VIP客户"; }

double getDiscountRate() const override {
    if (vipLevel == 1) return 0.95;
    if (vipLevel == 2) return 0.90;
    return 0.85;  // level 3
}

int getMaxActiveBookings() const override {
    if (vipLevel == 1) return 5;
    if (vipLevel == 2) return 8;
    return 10;  // level 3
}
```

**特有方法：**
- `upgradeLevel()`：根据累计消费自动升级VIP等级

**设计说明：**
通过虚函数，BillingManager在计算账单时无需判断客户类型，直接调用 `customer->getDiscountRate()` 即可得到正确折扣。这就是多态的威力。

---

### 4.3 `Booking` 预订记录类

**作用：** 封装一次完整的预订信息，包括客户、房间、入住退房日期、实际入住退房日期、当前状态等。

**核心成员：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `bookingId` | `string` | 预订唯一编号，如"B001" |
| `customerId` | `string` | 客户ID |
| `roomNumber` | `string` | 房间号 |
| `checkInDate` | `Date` | 预计入住日期 |
| `checkOutDate` | `Date` | 预计退房日期 |
| `actualCheckIn` | `Date` | 实际入住日期（可能为空） |
| `actualCheckOut` | `Date` | 实际退房日期（可能为空） |
| `status` | `enum` | PENDING/CHECKED_IN/CHECKED_OUT/CANCELLED |

**核心方法：**

```cpp
int calculateNights() const;        // 计算预计住几晚
bool isDateConflict(Date, Date) const;  // 判断某日期区间是否与本预订冲突
void checkIn(const Date& today);    // 状态变为CHECKED_IN
void checkOut(const Date& today);   // 状态变为CHECKED_OUT
void cancel();                      // 状态变为CANCELLED
void extendStay(const Date& newCheckOut);  // 延长退房日期
```

**运算符重载：**
- `operator==`：按bookingId比较
- `operator<`：按入住日期排序
- `operator<<`：输出预订详情

**设计说明：**
Booking类是连接客户、房间、账单的中心。它的状态转换（PENDING→CHECKED_IN→CHECKED_OUT）对应酒店的核心业务流程。

---

### 4.4 `BookingException` 预订异常类

**作用：** 在预订流程中遇到业务规则违反时抛出异常，让上层统一处理。

**继承：** `class BookingException : public exception`

**错误码设计：**

```cpp
enum Code {
    ROOM_NOT_FOUND = 1001,           // 房间不存在
    ROOM_UNAVAILABLE = 1002,         // 房间不可用
    CUSTOMER_NOT_FOUND = 1003,       // 客户不存在
    DATE_INVALID = 1004,             // 日期无效
    DATE_CONFLICT = 1005,            // 日期冲突
    BOOKING_NOT_FOUND = 1006,        // 预订不存在
    ALREADY_CHECKED_IN = 1007,       // 已入住
    ALREADY_CHECKED_OUT = 1008,      // 已退房
    MAX_BOOKINGS_EXCEEDED = 1009,    // 超过最大预订数
    BOOKING_CANCELLED = 1010         // 预订已取消
};
```

**方法：**
- `what()`：覆盖父类方法，返回错误信息
- `getErrorCode()`：返回错误码

**使用场景：**
在 `BookingManager::createBooking` 中，任何检查失败都可以抛出对应异常。例如：
```cpp
if (room == nullptr) {
    throw BookingException("房间不存在", BookingException::ROOM_NOT_FOUND);
}
```

### 4.5 `BookingManager` 预订管理器（核心业务）

**作用：** 处理酒店最核心的业务流程：客户注册、预订创建、入住、退房、续住等。

**内部容器：**
```cpp
vector<Customer*> customers;    // 多态存储普通客户和VIP客户
vector<Booking> bookings;       // 预订记录
RoomManager* roomMgr;           // 指向A的RoomManager
```

**核心方法详解：**

#### 客户管理
- `addCustomer(Customer* customer)`：注册客户，支持传入 `Customer*` 或 `VIPCustomer*`
- `findCustomer(id)`：按ID查找客户
- `getAllCustomers()`：获取所有客户
- `getVIPCustomers()`：获取VIP客户，内部使用 `dynamic_cast` 判断
- `upgradeCustomerToVIP(id, level)`：将普通客户升级为VIP

#### 预订创建 `createBooking`
这是最复杂的业务方法之一，需要依次检查：
1. 客户是否存在
2. 房间是否存在
3. 入住日期是否早于退房日期
4. 房间在指定日期是否可用
5. 客户当前活跃预订数是否超过上限
6. 创建Booking对象，修改房间状态为BOOKED

#### 入住 `checkIn`
1. 查找预订
2. 检查状态是否为PENDING
3. 设置实际入住日期
4. 将房间状态改为OCCUPIED

#### 退房 `checkOut`
1. 查找预订
2. 检查状态是否为CHECKED_IN
3. 设置实际退房日期
4. 将房间状态改为AVAILABLE
5. 返回Booking对象供BillingManager生成账单

#### 续住 `extendStay`
1. 查找预订
2. 检查新日期是否有效
3. 检查新日期是否与其他预订冲突
4. 修改checkOutDate

#### 查询方法
- `getCustomerBookings(id)`：获取某客户的所有预订
- `getRoomBookings(roomNumber)`：获取某房间的所有预订历史
- `getActiveBookings()`：获取当前活跃预订
- `getOverdueCheckOut(today)`：获取逾期未退房的预订

#### 文件IO
- `saveCustomers/loadCustomers`：保存/读取客户数据
- `saveBookings/loadBookings`：保存/读取预订数据

**内存管理：**
析构函数要 `delete` 所有 `Customer*` 指针。

## 5. 成员C：日期、账单与营收分析

成员C负责日期工具、账单生成和营收统计。这个模块对运算符重载和STL算法的要求最高。

### 5.1 `Date` 日期类

**作用：** 封装日期，提供日期的比较、加减、输入输出等操作。避免在系统中到处用字符串处理日期。

**为什么需要这个类：**
酒店系统中大量操作都涉及日期：预订入住日期、退房日期、账单日期、统计日期等。用专门的Date类可以让日期运算更直观、更安全。

**核心成员：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `year` | `int` | 年 |
| `month` | `int` | 月 |
| `day` | `int` | 日 |

**构造函数：**
- `Date()`：默认构造，可设为1900-01-01或当前日期
- `Date(int y, int m, int d)`：从年月日构造
- `Date(const string& dateStr)`：从"2026-06-13"字符串构造

**运算符重载（重点）：**

| 运算符 | 作用 |
|--------|------|
| `+` | 日期加N天 |
| `-` | 日期减N天，或两个日期相减得到天数差 |
| `++` / `--` | 日期自增/自减一天 |
| `<` `>` `==` `!=` `<=` `>=` | 日期比较 |
| `<<` / `>>` | 日期输入输出 |

**友元函数：**
```cpp
friend Date operator+(int days, const Date& d);  // 支持 5 + date 的写法
```

**核心方法：**
- `isValid()`：检查年月日是否合法
- `isLeapYear()`：判断闰年
- `getDaysInMonth()`：获取当月天数
- `toString()`：返回"YYYY-MM-DD"
- `static Date today()`：获取当前日期（AI辅助）

**实现难点：**
日期加减要考虑跨月、跨年、闰年。例如2026-01-31 + 1天 = 2026-02-01。

### 5.2 `Bill` 账单类

**作用：** 封装一次退房产生的账单，记录房费、服务费、折扣、总金额等信息。

**核心成员：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `billId` | `string` | 账单编号 |
| `bookingId` | `string` | 关联预订编号 |
| `customerId` | `string` | 客户编号 |
| `roomNumber` | `string` | 房间号 |
| `nights` | `int` | 住宿晚数 |
| `roomCharge` | `double` | 房费 |
| `serviceCharge` | `double` | 服务费 |
| `discountRate` | `double` | 折扣率 |
| `totalAmount` | `double` | 应付总金额 |
| `paid` | `bool` | 是否已支付 |
| `billDate` | `Date` | 账单生成日期 |

**核心方法：**

```cpp
void generate(const Booking& booking, const Room& room, const Customer& customer,
              double serviceRate = 0.1);
```
内部逻辑：
1. 用 `Date` 计算实际入住到退房的晚数
2. 房费 = 晚数 × 房间实际价格
3. 服务费 = 房费 × serviceRate
4. 折扣 = 客户折扣率（`customer.getDiscountRate()`）
5. 总金额 = (房费 + 服务费) × 折扣

```cpp
void applyDiscount(double rate);    // 额外折扣
void pay();                         // 标记已支付
```

**运算符重载：**
- `operator+`：合并两个账单金额
- `operator<<`：输出账单明细
- `operator double()`：类型转换，返回总金额，方便比较和统计

---

### 5.3 `BillingManager` 账单管理器

**作用：** 管理所有账单，提供账单CRUD、支付处理和营收统计分析。

**内部容器：**
```cpp
vector<Bill> bills;
int nextBillId;
```

**核心方法分类：**

#### 账单CRUD
- `generateBill(booking, room, customer)`：根据Booking生成Bill
- `processPayment(billId)`：处理支付，将账单标记为已付
- `findBill(billId)`：查找账单
- `getUnpaidBills()`：获取未付账单

## 6. 成员D：日志、工具与系统集成

成员D负责写通用工具类和系统门面。这部分代码逻辑相对直接，但要把整个系统串起来。

### 6.1 `Logger` 日志类（单例模式）

**作用：** 记录系统运行过程中的关键操作和异常信息，便于调试和审计。

**为什么用单例：**
整个系统只需要一个日志文件和一个日志写入器。如果创建多个Logger实例，会导致文件竞争和日志混乱。

**单例实现要点：**
```cpp
class Logger {
private:
    static Logger* instance;        // 唯一实例指针
    Logger();                        // 私有构造函数
    Logger(const Logger&) = delete;  // 禁止拷贝
    Logger& operator=(const Logger&) = delete;  // 禁止赋值
    
public:
    static Logger& getInstance();    // 获取唯一实例
};
```

**日志级别：**
```cpp
enum class Level { INFO, WARNING, ERROR };
```

**核心方法：**
- `log(level, message)`：写入一条日志
- `info(message)` / `warning(message)` / `error(message)`：便捷方法
- `getRecentLogs(n)`：获取最近n条日志
- `clearLogs()`：清空日志

**运算符重载：**
```cpp
void operator()(const string& message);           // 默认INFO级别
void operator()(Level level, const string& message);
```
这样可以使用 `logger("系统启动")` 的简洁写法。

**日志格式示例：**
```
[2026-06-13 14:30:05] [INFO] 系统初始化完成
[2026-06-13 14:31:12] [INFO] 客户C001预订了房间101
[2026-06-13 14:32:08] [ERROR] 房间201不存在
```
